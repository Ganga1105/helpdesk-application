# Module 4: File & Attachment Management (Iyyapan / Bharath)

### Iyyapan's Presentation Content (Frontend)
In my work on the File & Attachment module, I focused on creating a seamless and secure interface for users to share critical documentation within their support tickets. I implemented a modular and user-friendly upload system within the Ticket Details page, featuring a stylized attachment trigger that integrates with the browser's native file selection capabilities. To ensure a modern user experience, I leveraged the 'FormData' API to handle asynchronous file transmissions, providing users with immediate visual feedback through toast notifications for both successful uploads and validation errors. I also designed a dedicated 'Attachments' tab that serves as a centralized repository for all files associated with a ticket, complete with descriptive file metadata and direct download links. To maintain a clean and professional layout, I ensured that the file management interface remains secondary to the conversation flow while still being easily accessible whenever documentation is required. My implementation effectively bridges the gap between the user's local file system and our backend storage, ensuring that every attachment is handled with precision and displayed clearly within the support lifecycle.

### Bharath's Presentation Content (Backend)
For the backend implementation of Module 4, my primary goal was to architect a secure and efficient file storage and retrieval system that integrates perfectly with our ticketing logic. I developed a dedicated attachment management route that utilizes the Multer middleware to handle multi-part form data processing and secure disk storage. I implemented several layers of security, including a strict whitelist for file extensions—supporting everything from images and PDFs to specialized document formats—and a hard limit of 10MB per file to protect our server resources. Beyond physical storage, I engineered the metadata layer using Prisma to record critical information such as the original file name, the generated secure path, and the identity of the uploader, creating a permanent link between every file and its parent ticket. I also incorporated rigorous access control checks to ensure that customers can only upload or download files related to their own tickets, while maintaining full visibility for our support agents and administrators. Finally, I configured our Express server to serve these files through a protected static route, ensuring that every attachment is both secure and easily accessible through our frontend utility.
